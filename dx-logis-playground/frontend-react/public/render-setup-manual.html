  • VS Code (권장) — code.visualstudio.com

Python 설치 시 "Add to PATH" 옵션을 반드시 체크하세요. 안 하면 터미널에서 python 명령이 동작하지 않습니다.

GitHub 레포 구성

프로젝트 폴더 구조를 잡고 GitHub에 연결합니다

📁 권장 폴더 구조

  ├── frontend-vue/       # Vue.js 코드

  ├── .gitignore           # 제외 파일 목록
  └── README.md

TERMINAL

mkdir my-project
cd my-project

# 2. Git 초기화
git init

# 3. 하위 폴더 생성
mkdir backend frontend-vue frontend-react

📄 .gitignore 파일 생성

프로젝트 루트(my-project/)에 .gitignore 파일을 만들고 아래 내용을 붙여넣습니다.

.env
venv/
.venv/

# Node
node_modules/
dist/
.env.local
.env.production

# OS
.DS_Store

🐙 GitHub에 레포 만들기

  • github.com 접속 → 우측 상단 "+" → "New repository" 클릭
  • Repository name 입력 (예: my-project) → Public 또는 Private 선택 → "Create repository" 클릭
  • 생성된 레포 페이지에서 HTTPS URL 복사 (예: https://github.com/계정명/my-project.git)
TERMINAL — GitHub 연결 
# GitHub 레포와 연결 (URL은 본인 것으로 변경)
git remote add origin https://github.com/계정명/my-project.git
git branch -M main

# 첫 커밋
git add .
git commit -m "init: project structure"
git push -u origin main

Push 후 GitHub 웹에서 파일들이 보이면 정상입니다. 이후부터는 git push만 해도 Render가 자동 배포를 시작합니다.
STEP 03

FastAPI 백엔드 구성

Python 가상환경을 만들고 FastAPI 기본 구조를 잡습니다

TERMINAL — backend 폴더에서 실행 
# backend 폴더로 이동
cd backend

# Python 가상환경 생성 & 활성화
python -m venv venv

# Windows
venv\Scripts\activate
# Mac / Linux
source venv/bin/activate

# 패키지 설치
pip install fastapi uvicorn[standard] sqlalchemy asyncpg psycopg2-binary python-dotenv

📄 필수 파일 생성

backend/ 폴더 안에 아래 파일들을 만듭니다.

backend/main.py 
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
import os

app = FastAPI(title="My API", version="1.0.0")

# CORS 설정 — 프론트엔드 도메인 허용
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],  # 배포 후 실제 도메인으로 변경
    allow_methods=["*"],
    allow_headers=["*"],
)

@app.get("/")
async def root():
    return {"message": "FastAPI is running 🚀"}

@app.get("/health")
async def health():
    return {"status": "ok"}
backend/requirements.txt — 패키지 목록 저장 
pip freeze > requirements.txt
💡
requirements.txt는 Render가 배포 시 자동으로 읽어서 패키지를 설치합니다. 반드시 생성해야 합니다.

🧪 로컬 실행 확인

TERMINAL 
uvicorn main:app --reload

브라우저에서 http://localhost:8000 접속 → {"message": "FastAPI is running 🚀"} 보이면 성공

API 문서 자동 생성: http://localhost:8000/docs

STEP 06

Supabase — PostgreSQL DB 설정

Render DB 대신 Supabase 무료 플랜을 사용합니다 — 무기한 무료

🆚 Supabase vs Render PostgreSQL

Supabase Free ✅ Render Free
기간 무기한 90일 후 삭제
용량 500MB 256MB
대시보드 UI ✅ 풍부 (Table Editor, SQL Editor) 기본
비용 무료 무료 → $7/월

① Supabase 프로젝트 생성

  • supabase.com 접속 → GitHub 계정으로 가입/로그인
  • "New Project" 클릭
  • 설정 입력:
    • Name: my-project
    • Database Password: 강력한 비밀번호 입력 → 반드시 메모장에 저장
    • Region: Northeast Asia (Seoul) 선택
  • "Create new project" 클릭 → 약 2분 대기

② 연결 URL 복사

  • 프로젝트 대시보드 상단 "Connect" 버튼 클릭
  • 팝업에서 "Transaction pooler" 탭 선택
  • 아래 형태의 URI 복사 → 메모장에 저장
복사할 URL 형태 
postgresql://postgres.[project-ref]:[YOUR-PASSWORD]@aws-0-ap-northeast-2.pooler.supabase.com:6543/postgres
⚠️
URL 안의 [YOUR-PASSWORD]를 ①에서 설정한 비밀번호로 교체하세요. URL에 [ ] 대괄호가 남아있으면 연결 오류가 발생합니다.
⚠️
Supabase 무료 플랜은 1주일 미접속 시 프로젝트 일시정지됩니다. 주 1회 이상 대시보드에 접속하거나, UptimeRobot으로 주기적 헬스체크를 걸어두면 유지됩니다.

③ Supabase 대시보드 주요 기능

🗄️ 웹 UI에서 바로 DB 관리 가능
  • Table Editor — GUI로 테이블 생성 및 데이터 조회/수정
  • SQL Editor — SQL 직접 실행, 쿼리 저장
  • Database → Backups — 자동 백업 확인 (유료 플랜)
STEP 07

Render — FastAPI 배포

백엔드 API 서버를 Render Web Service로 배포합니다

  • 대시보드 → "New +" → "Web Service"
  • "Connect a repository" 선택 → GitHub 연동 (첫 번째라면 GitHub 앱 설치 허용) → my-project 레포 선택
  • 설정 입력:
    • Name: my-project-api
    • Region: Singapore
    • Branch: main
    • Root Directory: backend ← 꼭 입력!
    • Runtime: Python 3
    • Build Command: pip install -r requirements.txt
    • Start Command: uvicorn main:app --host 0.0.0.0 --port $PORT
    • Plan: Free
  • Environment Variables 섹션에서 "Add Environment Variable" 클릭:
Key Value 설명
DATABASE_URL 6단계에서 복사한 Supabase Transaction pooler URL Supabase PostgreSQL 연결 문자열
PYTHON_VERSION 3.11.0 Python 버전 고정
  • "Create Web Service" 클릭 → 배포 시작 (약 2~4분)
  • 배포 완료 후 상단에 표시되는 URL 메모: https://my-project-api.onrender.com
  • 브라우저에서 https://my-project-api.onrender.com/health 접속 → {"status": "ok"} 확인
✅
이후 git push를 하면 Render가 자동으로 새 버전을 감지하여 재배포합니다. CI/CD 별도 설정 없이 자동 적용됩니다.
STEP 08

Render — Vue.js 배포

Vue.js 앱을 Static Site로 배포합니다

  • 대시보드 → "New +" → "Static Site"
  • 같은 my-project 레포 선택
  • 설정 입력:
    • Name: my-project-vue
    • Branch: main
    • Root Directory: frontend-vue
    • Build Command: npm install && npm run build
    • Publish Directory: dist
  • Environment Variables 추가:
    • Key: VITE_API_URL
    • Value: https://my-project-api.onrender.com (7단계에서 메모한 URL)
  • "Create Static Site" 클릭 → 약 1~2분 배포
  • 배포 완료 후 URL 확인: https://my-project-vue.onrender.com
💡
Static Site는 완전 무료이며 슬립 없음. 무료 티어에서 느린 건 백엔드(Web Service)뿐입니다. Vue/React 프론트는 빠르게 로드됩니다.

서비스 연결 확인

🔗 FastAPI에서 CORS 도메인 업데이트

배포된 Vue/React URL을 backend/main.py의 CORS 설정에 추가합니다.

backend/main.py — CORS 수정

    CORSMiddleware,
    allow_origins=[
        "https://my-project-vue.onrender.com",
        "https://my-project-react.onrender.com",
        "http://localhost:5173",   # Vue 로컬 개발
        "http://localhost:5174",   # React 로컬 개발
    ],
    allow_methods=["*"],
    allow_headers=["*"],
)

STEP 11

CI/CD 흐름 확인

코드 변경 → 자동 배포까지의 흐름을 확인합니다

CI/CD 자동 배포 흐름
로컬에서 코드 수정
↓ git add . && git commit -m "..." && git push
GitHub main 브랜치 업데이트
↓ Webhook 자동 트리거
Render 변경 감지
↓ 변경된 서비스만 재빌드
FastAPI 재배포
Vue 재배포
React 재배포
✅ 약 2~4분 후 자동 반영 완료
💡
Render는 변경된 서비스만 재배포하지 않습니다. my-project 레포에 push가 발생하면 연결된 3개 서비스 모두 재빌드됩니다. 이를 최적화하려면 레포를 서비스별로 분리하거나, Render의 "Auto-Deploy" 설정에서 특정 경로 변경 시만 배포하도록 구성할 수 있습니다.

✅ CI/CD 테스트 방법

테스트 — 아무 파일이나 수정 후 push 
# backend/main.py의 메시지 수정 후
git add .
git commit -m "test: CI/CD check"
git push origin main

# Render 대시보드에서 "Deploying..." 상태 확인
# 약 3분 후 변경사항이 실제 URL에 반영되는지 확인
STEP 12

트러블슈팅

자주 발생하는 문제와 해결 방법

🔴 FastAPI 배포 실패 — "No module named 'xxx'"
  • 가상환경 활성화 상태에서 pip freeze > requirements.txt 재실행
  • 생성된 파일 확인 후 다시 push
🔴 API 요청 시 CORS 에러 (브라우저 콘솔)
  • main.py의 allow_origins에 프론트엔드 URL이 정확히 입력되어 있는지 확인
  • URL 끝에 슬래시(/)가 있으면 제거 (예: https://xxx.onrender.com ← 슬래시 없음)
🔴 첫 API 요청이 30초 이상 걸림

정상입니다. Render 무료 티어는 15분 미사용 시 서버가 슬립 상태가 됩니다. 첫 요청 시 워밍업 시간이 필요합니다. 해결책: UptimeRobot (무료)에서 5분마다 헬스체크를 걸어두면 슬립 방지 가능.

🔴 Vue/React 빌드 실패 — "sh: vite: not found"
  • Build Command를 npm ci && npm run build로 변경
  • package.json이 frontend-vue/ 또는 frontend-react/ 안에 있는지 확인
  • Root Directory 설정이 올바른지 재확인
🔴 DATABASE_URL 연결 오류
  • Supabase 대시보드 상단 "Connect" 버튼 → Transaction pooler URL 재확인
  • URL에 [ ] 대괄호가 없는지 확인 — 있으면 제거
  • [YOUR-PASSWORD] 부분이 실제 비밀번호로 교체되었는지 확인
  • 간헐적 오류(prepared statement)가 나오면 database.py에 statement_cache_size=0 추가
다음 단계로 고려할 것들

🗄️ DB 백업 & 정지 방지

Supabase 무료 플랜은 1주일 미접속 시 일시정지됩니다. UptimeRobot(무료)에서 5분마다 /health 엔드포인트에 헬스체크를 걸어두면 정지를 방지할 수 있습니다. 유료 전환 시 자동 백업이 활성화됩니다.

🔐 환경변수 보안

API 키, 비밀번호는 절대 코드에 하드코딩하지 말고 Render의 Environment Variables에 등록하세요. .env 파일은 .gitignore에 포함되어 있는지 확인하세요.

🌏 커스텀 도메인

Render에서는 무료로 커스텀 도메인 연결이 가능합니다. 가비아 또는 Cloudflare에서 도메인 구매 후, Render Settings → Custom Domain에 추가하면 됩니다.
